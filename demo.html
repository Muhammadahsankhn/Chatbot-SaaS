Data Flows
Training Flow — How Bot Learns
1
Customer fills Manual Form or uploads a filePOST /ingest/manual or POST /ingest/upload
2
file_parser.py extracts plain textPDF pdfplumber · DOCX python-docx · XLSX openpyxl · CSV built-in
3
Text split into 500-word overlapping chunkschunk_size=500, overlap=50 words
4
embeddings.py converts each chunk to 384-dim vectorall-MiniLM-L6-v2 model (local, no API needed)
5
knowledge_base.py saves vectors + metadata to diskdb/knowledge/{userId}_vectors.npy · {userId}_metadata.json
Chat Flow — How Visitor Gets Answer
1
Visitor types message in widget on customer sitePOST /chat/message { apiKey, message, sessionId, history }
2
validate_key.py verifies API key and fetches userIdMiddleware checks MongoDB for valid key
3
Question embedded to 384-dim vectorSame all-MiniLM-L6-v2 model used at training time
4
Cosine similarity search returns top 4 matching chunksThreshold: 0.25 similarity score minimum
5
Groq Llama 3.3 70B generates reply using contextConversation saved to MongoDB · Reply returned to widget
